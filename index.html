Neural Network for Predicting Formation Properties in Oil & Gas Wells - Ongoing

Software Stack: Python, TensorFlow, Kaggle

Training a small neural net to predict rock properties such as porosity and permeability from well log data. The dataset I am training the neural net on is rougly 8000 lines of well log data from Kansas Geological Survey. This dataset contain data on resistivity, spontaneous potential, gamma ray, bulk density, neutron and density porosity data.

The aim is to use typical well log data like neutron and gamma ray to predict more complex rock properties like porosity and permeability that are more difficult and expensive to measure directly. By training a neural network on this data, I hope to be able to accurately predict these rock properties from the more easily obtainable well log data.

Pet Rescue: Autonomous Robot

Software Stack: C++, C, Python, KiCad, Onshape
Fabrication Stack: Waterjet, Laser Cutting, Milling, Turning (Lathe), 3D Printing

My ENPH 253 team (Luca Jones, Harchetan Chohan, Taichi Kamei) designed and built an autonomous rescue robot capable of navigating a complex obstacle course, identifying and retrieving ‚Äúpet‚Äù targets, and returning them safely to the home zone, all without human control.

The brain of our robot was an ESP32 microcontroller, it was controlled by a custom state machine built to manage the various modes of operation such as line following and pet decection

For line following, we used PID control in combination with infrared LEDs and phototransistors. Utilizing the low reflectance of the electrical tapethe robot was able to determine it position relative to the line. The PID algorithm then continuously adjusted the robots direction to keep in on track.

To identify the pets, our robot used a 2D time-of-flight sensors to detect the presence of pet targets. The time of flight sensor collected a discrete depth map of the robots surrondings, and by using the cylindrical nature of the pets could be distinguished from other objects.

Once we detected a pet we needed a way to pick up the pet. We fabricated a custom 2-link planar manipulator arm paired with a 4-bar linkage claw. To control the rotation of the arm, we combined a DC motor, magnetic encoder, and PID control, effectively creating our own servo to meet size contraints. The arm's bending motion and claw's closing action were controlled by standard 25kg servos.

There were a few strategies to return the pets to the safe area. We decided on the bucket-and-zipline strategy. To lift the bucket to the zipline, we designed a continuous cascade lifting mechanism driven by a DC motor that winds paracord around a spool

Document Translator

Software Stack: Python, Google Cloud SDK (Translate, Cloud Vision, Cloud Storage)

During my internship at Condor Energies, I developed a pipleine that automates the process of translating documents from Uzbek, Russian and Kazakh to English. For this project I had to handle a wide variety of document types such as scanned PDFs, Word, Excel and PowerPoint files. Each had its own challenges when it came to extracting text as well as writing the translated text back into the original format.

Scanned PDFs

This involved using Optical Character Recognition (OCR) to extract text from scanned documents such as old drilling reports and well logs. Then using the Google translate API to translate the extracted text before finally writing the translated text back into a new PDF that closely resembled the original document. A particular challenge with this task was ensuring the formatting and layout of the original document was preserved in the translated version. For most PDFs I was dealing with they were scanned documents, so in order to write the translated text back to the PDF I had to 'remember' the pixel location of each word in the original document and then write the translated text back to those same locations.

Word, Excel, PowerPoint

For these document types I used the python-docx, openpyxl and python-pptx libraries to extract text from the documents, translate them using the Google Translate API, and then write the translated text back into new documents. This was much more straightforward than dealing with scanned PDFs, but still required careful handling to ensure formatting and layout were preserved.

Production Forecasting Automation Tool

Software Stack: Microsoft VBA

During my internship at Condor Energies, I developed an automation tool to streamline the production forecasting process for natural gas wells. The tool ingests historical production data, cleans and preprocesses it, and then applies decline curve analysis to forecast future production rates. The tool generates field-level and well-level forecasts, complete with visualizations and summary reports. The tool successfully reduced about 3 days worth of work for a resevoir engineer down to just a few hours once per month.
